# codexschema

Pinned JSON Schema and generated Go types for the Codex app-server stdio JSON-RPC protocol.

## Pinned version

**codex-cli 0.133.0** (default output, no `--experimental` flag).
The machine-readable SSOT is `test-harness/tool-versions.json` under the
`codex.schema` role; the separate `codex.fidelity` pin intentionally tracks
the real CLI version used by FakeVsReal contracts.

## Package layout

| Path | Contents |
|---|---|
| `schema/codex_app_server_protocol.schemas.json` | v1/shared aggregate bundle (drift-detection reference) |
| `schema/codex_app_server_protocol.v2.schemas.json` | v2 aggregate bundle (drift-detection reference) |
| `v1/types.gen.go` | Generated Go types for v1 protocol (`package codexschemav1`) |
| `v2/types.gen.go` | Generated Go types for v2 protocol (`package codexschemav2`) |
| `methods.go` | Method-name constants and approval reply values (`package codexschema`) |

## Updating the pin

Run the Makefile target to regenerate both the schema bundles and Go types in one step:

```sh
scripts/install-pinned-codex.sh schema
make codex-schema-update
make codex-schema-check    # verify committed pin matches current codex output
```

The schema-update target:
1. Runs `codex app-server generate-json-schema --out <tmpdir>`
2. Copies the two aggregate bundle files to `schema/`
3. Re-runs quicktype to regenerate `v1/types.gen.go` and `v2/types.gen.go`
4. Commits should include the updated pin version in this README

## Drift detection (CI)

CI installs the `codex.schema` version and runs `generate-json-schema`, then diffs the two committed bundles.
A schema version bump requires an explicit PR that updates the pin and regenerates types.

## codegen tool

Types are generated by [quicktype](https://github.com/quicktype/quicktype) with `--src-lang schema`.
The generated files carry `// Code generated from JSON Schema using quicktype. DO NOT EDIT.` headers
and are excluded from golangci-lint's file-length and function-length checks automatically.

```sh
# regenerate v2 types manually
npx quicktype --lang go --package codexschemav2 --src-lang schema \
  -o v2/types.gen.go path/to/individual/v2/*.json

# regenerate v1 types manually
npx quicktype --lang go --package codexschemav1 --src-lang schema \
  -o v1/types.gen.go path/to/individual/v1/*.json
```
