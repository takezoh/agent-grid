# ADR 0021 — Hand-write the frontend wire types; verify with round-trip tests

Status: Accepted

## Context

A1-α landed a typed wire on the Go side (`client/proto/surface_command.go`
and `surface_event.go`). The β frontend must consume and produce this wire
in TypeScript. Two approaches:

1. **Code-gen**: a `go generate` directive that translates Go structs into
   TypeScript interfaces, e.g. via `tygo` or a homegrown script.
2. **Hand-write**: a TypeScript `wire/server.ts` + `wire/client.ts` that
   mirrors the Go shape, with round-trip tests against canonical JSON.

The wire surface for β is narrow (6 commands + 4–5 events). It is fixed by
A1-α and only grows in γ/δ (view-update fields, transcript tail). Codegen
adds a build-time dependency and a code review hop for what amounts to ~150
lines of TypeScript.

## Decision

Hand-write the TypeScript wire types in `src/client/web/src/wire/`. Add
round-trip tests that:

- Load canonical JSON fixtures (one per frame kind) generated by a small
  Go helper (`src/client/web/src/wire/testdata/`).
- Round-trip each fixture through the TS codec and assert `JSON.stringify`
  equality after normalization.
- A Go test (`src/client/web/wire_fixtures_test.go`) regenerates the
  fixtures and fails if they drift from the Go side. This keeps the two
  sources in lockstep without runtime codegen.

If the wire grows past ~500 lines of TS or > 30 frame kinds, revisit
codegen.

## Consequences

- No additional toolchain (`tygo` / `quicktype` / Go AST parser) in the
  build.
- Frontend reviewers can read the wire types directly.
- A drift between Go and TS shows up as a fixture diff in CI, not a silent
  runtime mismatch.
- Maintenance cost: when γ/δ add fields, they must be added in two places.
  The fixture test catches omissions.

## Alternatives

- **`tygo`** — popular Go → TS generator. Adds a Go module dependency and
  a build step; the diff in regenerated TS files is noisy in PRs.
- **`quicktype`** — generates TS from JSON Schema. Requires writing JSON
  Schema or running JSON through the tool; another build step.
- **Hand-written types without fixture tests** — risks silent drift between
  Go and TS shapes. Rejected.
